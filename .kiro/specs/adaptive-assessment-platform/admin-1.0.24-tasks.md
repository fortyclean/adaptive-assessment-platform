# مهام إصدار 1.0.24 - تحسين شاشات المشرف

## الهدف

إنهاء التحسينات المتبقية في شاشات المشرف دون تغيير الطابع البصري أو إعادة بناء الشاشات من الصفر، مع الحفاظ على تسجيل الدخول بكل صوره، وتحديث سجل الإصدارات داخل التطبيق.

## قائمة المهام التفصيلية

### 1. تفعيل أزرار إعدادات المؤسسة

- حفظ بيانات المؤسسة محلياً بشكل مستمر بدلاً من الاكتفاء برسالة مؤقتة.
- حفظ إعدادات العام الدراسي، الفصل الدراسي، مقياس التقييم، اللغة، المنطقة الزمنية، والتنبيهات.
- إبقاء أزرار التنقل الحالية كما هي: الفصول، المستخدمون، التنبيهات، الدعم، عن التطبيق، وتسجيل الخروج.
- معيار القبول: عند الرجوع للشاشة تبقى آخر القيم المختارة ظاهرة للمشرف.

### 2. تحسين إجراءات التقارير المدرسية

- إضافة إجراء واضح لتحديث بيانات التقارير من رأس الشاشة.
- إضافة ملخص للفلاتر الحالية حتى يعرف المشرف ما الذي يراه.
- إبقاء زر عرض الفصول موجهاً إلى شاشة إدارة الفصول.
- معيار القبول: لا تظهر أزرار بلا نتيجة، وكل إجراء يعطي تحميل/تحديث/انتقال واضح.

### 3. تحسين تعديل المستخدمين وربط الفصول

- استبدال إدخال معرفات الفصول يدوياً بقائمة اختيار من الفصول المتاحة.
- دعم البحث داخل الفصول عند تعديل المستخدم.
- الإبقاء على التحديث عبر API في الإنتاج، وبيانات ديمو خارجية في حسابات التجربة.
- معيار القبول: يستطيع المشرف تعديل اسم المستخدم، بريده، والفصول المرتبطة من نفس النافذة.

### 4. تحسين ربط المعلمين والطلاب بالفصول

- إضافة بحث داخل قائمة المعلمين.
- إضافة بحث داخل قائمة الطلاب.
- إضافة تحديد الكل وإلغاء الكل للطلاب الظاهرين بعد البحث.
- معيار القبول: يستطيع المشرف الوصول للمعلم أو الطالب بسرعة دون تمرير طويل.

### 5. إضافة فحص دخان تقني للنسخة المنشورة

- إنشاء سكربت backend خفيف يفحص الصحة العامة، تسجيل الدخول، refresh، المستخدمين، الفصول، والتقارير.
- يعتمد على متغيرات بيئة ولا يحفظ أي أسرار داخل الكود.
- معيار القبول: يمكن تشغيله بالأمر `npm run smoke`.

### 6. تحديث الإصدار وسجل الإصدارات

- رفع الإصدار إلى `1.0.24+24`.
- تحديث سجل الإصدارات داخل التطبيق.
- تحديث دليل الاستخدام ودليل النشر.
- بناء APK باسم لا يحتوي على كلمة railway.
- معيار القبول: يظهر الإصدار 1.0.24 داخل شاشة "عن التطبيق" وتُبنى النسخة بنجاح.
